# Tools

CompilerSutra should not stop at articles. The tools layer is where readers turn static learning into interactive inspection.

## Planned Tools

### LLVM IR Visualizer

Inspect generated IR, follow control flow, and connect instructions to the source-level intuition behind them.

### Optimization Diff Viewer

Compare outputs across optimization levels, pass pipelines, or compiler choices such as GCC versus Clang.

### CFG and SSA Graph Generator

Visualize control-flow graphs, dominance, and SSA relationships so analysis concepts stop feeling abstract.

### Pass Pipeline Explorer

Track how a sequence of passes changes IR over time instead of treating the optimizer like a black box.

### Benchmark Artifact Explorer

Browse benchmark JSON, CSV, and comparison outputs with enough structure to spot meaningful differences fast.

## How to Use This Section

Use tools after reading a core article, not before. The right loop is:

1. learn the concept
2. inspect a concrete example
3. compare before and after states
4. write down what actually changed
