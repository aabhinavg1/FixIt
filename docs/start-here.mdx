## Roadmap

### Stage 1: Compiler Foundations

Goal: understand the compiler pipeline well enough to reason about frontend, IR, and backend responsibilities.

- [What Is a Compiler?](/docs/compilers/Compiler)
- [Intermediate Representation in Compilers](/docs/compilers/ir_in_compiler)
- [From Source Code to Executable](/docs/compilers/sourcecode_to_executable)
- [Introduction to Compilers](/docs/compilers/IntroductionToCompilers)
- [Compiler Frontend](/docs/compilers/front_end/compiler-frontend)

Checkpoint:
You should be able to explain how source code moves through parsing, IR construction, optimization, and code generation.

### Stage 2: LLVM and IR

Goal: become comfortable with LLVM as a production-grade compiler infrastructure.

- [LLVM Roadmap](/docs/llvm/intro-to-llvm)
- [What Is LLVM?](/docs/llvm/llvm_basic/What_is_LLVM)
- [LLVM Architecture](/docs/llvm/llvm_basic/LLVM_Architecture)
- [LLVM IR Introduction](/docs/llvm/llvm_ir/intro_to_llvm_ir)
- [Hierarchy of LLVM IR](/docs/llvm/llvm_ir/hierarchy_of_llvm_ir)
- [Manage LLVM Versions](/docs/llvm/llvm_extras/manage_llvm_version)

Checkpoint:
You should be able to read simple LLVM IR, run LLVM tools, and understand where passes fit.

### Stage 3: Optimization and Passes

Goal: understand how optimizers reason about programs and how passes transform IR safely.

- [Static Single Assignment](/docs/llvm/llvm_Curriculum/level0/Static_Single_Assignment)
- [SSA Part 2](/docs/llvm/llvm_Curriculum/level0/Static_Single_Assignment_part2)
- [Dominator Tree and Dominance Frontier](/docs/llvm/llvm_Curriculum/level0/Dominator_Tree_And_Dominance_Frontier)
- [LLVM Pass Tracker](/docs/llvm/llvm_pass_tracker/llvm_pass)
- [Compiler Flags](/docs/compilers/flag/compiler-flags)

Checkpoint:
You should be able to explain why SSA exists, why dominance matters, and how a pass pipeline changes code quality.

### Stage 4: GPU and Heterogeneous Compilation

Goal: connect compiler knowledge with parallel hardware and GPU execution constraints.

- [What Is GPU?](/docs/gpu/what_is_gpu)
- [OpenCL](/docs/gpu/opencl)
- [Setting Up OpenCL](/docs/gpu/opencl/basic/setting_up_opencl)
- [Writing Your First Kernel](/docs/gpu/opencl/basic/Writing_your_first_kernel)
- [GPU Programming TOC](/docs/gpu/gpu_programming/gpu_programming_toc)

Checkpoint:
You should be able to explain the relationship between compiler choices, kernel behavior, and device execution.

### Stage 5: ML and Domain-Specific Compilers

Goal: understand how modern compiler stacks support AI workloads and custom dialects.

- [MLIR Intro](/docs/MLIR/intro)
- [TVM Intro](/docs/tvm/intro-to-tvm)

Checkpoint:
You should be able to describe why multi-level IR exists and why ML compilers need staged lowering.

## Use the Site Like a Product

The strongest way to use CompilerSutra is:

1. Start with a track.
2. Read the core article for that stage.
3. Move into a related lab or project.
4. Use a tool to inspect IR, passes, graphs, or benchmark results.
5. Come back for the next stage instead of browsing randomly.
